

# Make HA Add-on Self-Contained

## Problem
HA builds only from `bjorq_asset_wizard/` вЂ” the `server/` source code lives at repo root and is not included in the build context.

## Approach
Expand the existing `prepare-addon.sh` script to stage all required files, update the Dockerfile to build the server properly from staged files, and align ports between config/run.sh/Dockerfile.

## Changes

### 1. `bjorq_asset_wizard/prepare-addon.sh` вЂ” Expand staging
Copy all required files into the add-on directory:
- `server/` (source, package.json, tsconfig.json)
- Root `package.json` and `package-lock.json` are NOT needed вЂ” the server has its own

### 2. `bjorq_asset_wizard/Dockerfile` вЂ” Proper server build
```dockerfile
ARG BUILD_FROM=ghcr.io/home-assistant/amd64-base:3.19
FROM ${BUILD_FROM}

RUN apk add --no-cache nodejs npm python3 make g++ vips-dev

WORKDIR /app

# Copy staged server source
COPY server/package.json server/package-lock.json* ./
RUN npm ci --ignore-scripts && npm rebuild sharp

COPY server/tsconfig.json ./
COPY server/src/ ./src/
RUN npx tsc

# Copy run script
COPY run.sh /run.sh
RUN chmod +x /run.sh

# Storage dirs created at runtime by run.sh
ENV NODE_ENV=production PORT=3000 HOST=0.0.0.0

EXPOSE 3000
CMD ["/run.sh"]
```

Key changes:
- Build only the server (not `npm run build` from root)
- Install `vips-dev` for sharp native module
- Output lands in `/app/dist/`
- Port aligned to 3000

### 3. `bjorq_asset_wizard/run.sh` вЂ” Align port and paths
- Change `PORT=3500` в†’ `PORT=3000`
- Change `exec node /app/dist/index.js` (already correct path after tsc builds to `/app/dist/`)

### 4. `bjorq_asset_wizard/.gitignore` вЂ” Ignore staged files
Add `server/` so the staged copy isn't committed (it's generated by prepare-addon.sh).

### 5. `bjorq_asset_wizard/config.yaml` вЂ” Version bump to 0.1.2
So HA detects the update.

### 6. `bjorq_asset_wizard/README.md` вЂ” Update docs
Explain the prepare в†’ build flow.
